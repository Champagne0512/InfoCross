核心特色与设计规范
1. 项目愿景 (Vision)
InfoCross 是一个基于 AI 语义分析 的校园跨学科信息聚合与协作平台。
它旨在打破高校中以“学院”为单位的信息孤岛，通过技术手段将学习、科研、竞赛与校园生活（娱乐/运动）有机融合，帮助学生探索“专业之外”的可能性。
2. 核心特色功能 (Core Unique Selling Points)
开发指令： 以下四个功能是本项目的灵魂，必须作为重点开发对象，技术实现需完全遵循所述逻辑。
🌟 特色一：基于语义的“破壁”推荐引擎 (The Wall-Breaker Engine)
核心逻辑： 拒绝同质化推荐。系统利用 Vector Database (向量数据库) 计算语义距离，故意推荐“用户专业之外、但在逻辑上高度相关”的内容。
场景示例： 系统识别到计算机系学生关注“深度学习”，不会只推编程比赛，而是跨界推送哲学系的讲座《人工智能的伦理困境》（语义强关联）。
UI 表现： 在推荐卡片上显著标注 [ 破壁推荐 ] 标签，并附带 AI 生成的推荐理由：“基于你的代码背景，推荐你补充伦理视角。”
🌟 特色二：AI “省流”摘要代理 (AI Smart Briefing Agent)
核心逻辑： 解决“红头文件/长通知”阅读难的问题。利用 LLM (大语言模型) 对非结构化文本进行清洗和提取。
提取维度： 自动从长文中抓取 [时间]、[地点]、[硬性门槛]（如：仅限大三）、[收益]（如：综测加分、现金奖励）。
UI 表现： 详情页顶部必须展示 "TL;DR" (太长不看) 模块，以结构化卡片形式展示关键信息，而非纯文本。
🌟 特色三：“技能互补型”智能组队 (Skill-Match Team Builder)
核心逻辑： 改变“熟人组队”模式。系统像猎头一样，基于技能互补原则进行撮合。
匹配算法： 当队长发布“互联网+比赛”招募时，AI 分析项目需求，识别出团队缺乏“财务分析”和“UI设计”角色，优先向具备这些技能的非本专业用户推送邀请。
UI 表现： 组队大厅展示“技能缺口可视化图表”，而非简单的文字列表。
🌟 特色四：校园“探索者”可视化足迹 (Explorer Footprint)
核心逻辑： 用数据证明学生生活的丰富性。
数据可视化： 实时计算用户的活动分布（学术 / 娱乐 / 运动 / 艺术 / 公益），生成动态雷达图。
成就体系： 当理科生参与人文活动，解锁 [ 文理兼修 ] 勋章；当活动地点跨越多个校区，解锁 [ 校园旅行家 ] 勋章。所有勋章均为矢量图形设计。
3. UI 设计规范： "Vivid Rationality" (生动理性)
设计原则：
NO EMOJI: 全面禁止使用 Emoji。所有图形元素必须使用高质量的 SVG 矢量图标 (推荐 Phosphor Icons 或 Lucide Icons)。
Color Coded (色彩编码): 用颜色区分功能，而非装饰。
High Density (高密度): 像仪表盘一样展示信息，体现专业感。
3.1 色彩系统 (Color System)
保持背景干净，用高饱和度的功能色点亮界面。
Canvas (底色): #FAFAFA (极浅灰白，类似高级纸张)。
Surface (卡片): #FFFFFF (纯白，带微弱弥散阴影)。
Text (文字):
Primary: #0F172A (深蓝黑，理性)
Secondary: #64748B (蓝灰，清晰)
Category Colors (功能色 - 必须严格对应):
🔵 科研/比赛 (Academic): #2563EB (Royal Blue)
🟠 娱乐/生活 (Life): #F97316 (Vibrant Orange)
🟢 运动/公益 (Activity): #059669 (Emerald Green)
🟣 AI/跨界 (Intelligence): #7C3AED (Electric Violet) - 专用于 AI 推荐和破壁功能。
3.2 字体系统 (Typography)
Headings (标题): Outfit 或 Plus Jakarta Sans。几何感强，现代、圆润、友好。
Body (正文): Inter。高可读性标准字体。
Data/Tags (数据/标签): JetBrains Mono 或 Space Mono。等宽字体，用于展示时间、人数、AI 算力值，增加“极客感”。
3.3 图标系统 (Iconography)
风格: Duotone (双色风格)。
规范: 主线条为深灰色，填充部分为半透明的主题色（如半透明的蓝）。
示例:
Trend (趋势) -> 使用 ChartLineUp 图标。
AI (智能) -> 使用 Sparkle 或 MagicWand 图标。
Team (组队) -> 使用 UsersThree 图标。
4. 页面布局与交互 (Layout & Interaction)
4.1 首页：The Dashboard (控制台风格)
抛弃传统的博客列表，采用 Bento Grid (便当盒) 布局，营造丰富的信息层级。
Header (顶部):
左侧：问候语（大号字体）+ 日期（等宽字体）。
右侧：数据磁贴。展示 [今日新增: 12] [破壁指数: 89%]，背景为极淡的灰色，文字为单色。
Navigation (导航):
使用 Pill Shape (胶囊形) 选项卡切换：[全部] [学术] [生活] [破壁推荐]。选中项为黑色背景白字，未选中为灰色文字。
Content Grid (内容网格):
卡片样式: 白色大圆角卡片，无边框，悬浮阴影。
视觉降噪: 对于没有海报的活动，自动生成基于类别色的纯色背景 + 巨大的半透明矢量图标作为封面。
社交堆叠: 卡片右下角显示 [User Icon] [User Icon] +32，表示参与人数，使用圆形头像剪切。
AI 推荐理由: 卡片底部附带紫色背景条，显示 [Icon: Sparkle] 推荐理由: 匹配你的摄影兴趣。
4.2 详情页：The Flyer (电子传单风格)
双栏布局 (6:4):
Left Column (内容区):
排版宽松，类似杂志。
Highlight: AI 提取的重点句子，使用荧光笔效果（文字下方叠加半透明色块）高亮显示。
Right Column (侧边栏 - 悬浮):
TL;DR Card: 位于最上方。用紫色边框包裹，内部使用等宽字体列出核心信息（时间、地点、门槛）。
Skill Graph: 一个小型的雷达图，展示参加此活动需要的能力维度（如：逻辑 80%，体力 20%）。
Action: 底部固定巨大的黑色按钮 "Join Event"。
4.3 组队/发布页：The Canvas (画布风格)
便利贴墙: 每一条组队需求设计成一张正方形的卡片，背景色使用极淡的类别色（淡蓝、淡黄）。
技能缺口: 在卡片上用虚线圆圈表示待填补的角色，例如 ( ) UI设计师，点击即可申请填补该空缺。
实时预览: 在发布表单右侧，实时渲染出卡片最终在首页显示的样子，所见即所得。
4.4 个人中心：Digital ID (数字身份)
ID Card: 顶部展示一张类似未来工牌的卡片。包含头像、姓名、学院、以及一个装饰性的条形码。
Data Viz (数据可视化):
Radar Chart: 展示用户的“破壁维度”（文/理/工/艺）。
Badge Wall: 勋章墙。获得的勋章以精美的矢量徽章形式展示，未获得的显示为灰色轮廓。
5. 动效规范 (Motion) - 微交互
Hover: 鼠标悬停在卡片上时，卡片向上平移 4px，阴影扩散，同时封面图轻微放大 (Scale 1.05)。
Click: 按钮点击时，产生 0.95 的缩放回弹效果。
Loading: 使用 Skeleton (骨架屏) 配合呼吸灯效果，禁止使用传统的旋转圆圈。
💡 给开发的最终执行建议
字体先行: 在 CSS 中首先引入 Outfit, Inter, JetBrains Mono。字体对了，高级感就出来了一半。
严控颜色: 在 tailwind.config.js 中锁死上述色值，严禁开发者随意取色。
图标替换: 检查所有代码，凡是出现 Emoji 的地方，全部替换为 Phosphor Icons 的 SVG 组件。
AI 存在感: 确保所有 AI 生成的内容（摘要、推荐理由）都使用紫色系视觉元素进行区分，强化“智能助手”的感知。
