## InfoCross 项目展示速记

### 1. 项目初衷（Why）
- 校园活动/讲座/竞赛情报分散在教务处、公众号、班群等渠道，信息极度割裂，学生需要花大量时间筛选。
- 推荐算法造成“信息茧房”，理工科同学难以看到艺术/人文机会，也错过跨学科合作的可能。
- 活动通知动辄几千字，阅读成本高，学生希望有一个自动化的“摘要 + 真正想要的推荐”入口。
- InfoCross 因此定位为“校园跨学科信息聚合与破壁平台”，以 Supabase + Vue 3 + AI 能力为核心，把散落的数据拉通，让不同学院的人快速建立连接。

展示时可强调：我们做的不是又一个资讯流，而是“聚合 + AI 摘要 + 向量破壁 + 团队协作”一体化体验，解决了“看不到 + 看不懂 + 找不到队友”的痛点。

### 2. 项目内容（What）
- 前端：Vue 3 + TypeScript + Vite，配 Pinia、Vue Router、Tailwind（Nexus Design System 风格），提供 Focus / Vibe 双模式的 UI 体验。
- 后端：Supabase 负责 Auth、Postgres、Storage、Realtime，利用 RLS 保障行级安全，pgvector 支撑向量检索。
- AI 侧：自研 Edge Function `analyze-article` 调用大模型完成 OCR、摘要、标签和 embedding 生成；`recommend` Function 用余弦相似度输出“破壁推荐”。
- 数据域：包含文章资讯、互动（收藏/点赞/加入）、团队招募与小组协作（申请、临时沟通、聊天室、任务、文件）。

可在展示中配合系统架构图：浏览器——Vite 前端——Supabase 服务（Auth/DB/Storage/Edge Functions），突出轻量 Serverless 架构和 AI 插件化能力。

### 3. 核心功能（How）
1. **信息聚合 Hub**：
   - 社团/组织可上传活动海报、标题、正文，系统支持中文/英文信息混排。
   - 前端触发 Edge Function 自动完成 OCR + 摘要 + 标签，写回 `articles` 表；列表页按学院、类型、标签筛选。
2. **AI 破壁推荐**：
   - 用户档案里维护兴趣标签 + 模式（Focus 强检索、Vibe 灵感发现）。
   - 系统将标签 embed 成向量，与文章向量做余弦相似度排序，给出跨院系推荐；同屏展示 AI 理由和标签。
3. **互动与通知**：
   - 点赞、收藏、报名（`interactions` 表）驱动个性化资讯，Realtime 通知在右上角 Inbox 显示。
4. **团队协作闭环**：
   - 招募详情页支持“未加入成员 ⇄ 队长”的临时沟通（`team_interest_messages`），帮助筛选合适的潜在成员。
   - 正式成员进入 Workspace，可使用聊天室（`team_chat_messages`）、任务板（`team_tasks` + 指派）、文件库（`team_files`）。
   - 申请流程（`team_applications`）与审批策略通过 Supabase RLS 控制，只允许队长/管理员处理。
5. **运营工具**：
   - Docs 中提供需求、UI、工程结构规范，保证演示前可以快速校准设计语言与功能范围。

展示时建议带着真实 demo 路径：1）上传活动海报 → 展示 AI 自动填表 → 2）切换到推荐页看破壁结果 → 3）打开团队详情发起临时沟通 → 4）转到 Workspace 看任务/文件动态。

### 4. 项目优势（Value）
- **真实痛点驱动**：紧扣“信息分散 + 跨学科合作难”两大痛点，目标用户清晰，价值叙事很直接。
- **AI 深度融合**：不仅“加个聊天机器人”，而是把 OCR、摘要、标签、向量检索嵌入业务链路，显著降低运营成本。
- **全链路体验**：从信息发现 → 意向沟通 → 正式协作（任务、文件、实时群聊）一站式完成，减少各平台切换。
- **安全与可扩展性**：Supabase + RLS 架构保证敏感数据只对相应角色开放，Edge Functions 方便扩展更多算法或第三方 API。
- **设计规范化**：Tailwind + Nexus Design System + docs 中的 UI/工程规范，确保演示版与交付版一致且易维护。

### 5. 展示话术小贴士
1. **开场 30s**：用“校园信息破壁机”这个比喻，加上痛点例子（错过跨院黑客松/讲座）抓住评审注意力。
2. **目标用户**：强调覆盖学生、社团负责人、辅导员三类角色，说明平台如何同时满足他们。
3. **功能走查**：严格按上述 Demo 路径，配合实时数据（Supabase seed）防止“空白状态”。
4. **数据与 RLS**：提一句我们实现了行级安全策略（如 `team_interest_messages` 的双向策略），展现对安全和权限的重视。
5. **收尾**：给未来规划（接入更多校园 API、开放 AI 摘要 API），突出可持续发展。

照此讲述即可完整呈现“初衷-内容-功能-优势”，且不会夸大不存在的特性。
